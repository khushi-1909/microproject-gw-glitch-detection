# GW Glitch Detection: Blip vs White-Noise Burst Classification

A 1D CNN classifier that distinguishes **blip glitches** from **white-noise burst (WNB) signals** in simulated single-detector gravitational-wave strain data.

Built as part of a research microproject at Cardiff University's School of Physics and Astronomy, under the supervision of Dr Vassilis Skliris.

---

## Problem

Ground-based gravitational-wave detectors like LIGO produce strain data contaminated by short-duration non-Gaussian transients called **glitches**. Blip glitches are one of the most common and problematic glitch classes — they are morphologically similar to real GW burst signals and can elevate false-alarm rates in low-latency search pipelines.

This project poses a **binary classification problem**:

- **Class 0:** noise + white-noise burst (WNB) — proxy for a real GW burst signal
- **Class 1:** noise + blip glitch — transient noise artefact

The goal is to train a lightweight CNN to separate these two classes, with performance evaluated using the **False Positive Rate (FPR)** and **AUC-ROC** across a range of signal-to-noise ratio (SNR) bins.

---

## Method

### Data Generation

- **WNB waveforms** generated using the [MLy](https://git.ligo.org/cbc/mly) framework with randomised frequency bounds (30–400 Hz) and durations (0.05–0.5 s)
- **Blip glitches** generated using [gengli](https://github.com/MelissaLopez/gengli) and injected into optimal Gaussian background noise via MLy
- 4000 examples per class; SNR drawn uniformly from [5, 50]
- All segments: 1 s duration, 1024 Hz sampling rate, single detector

### Feature Representation

Each strain segment is standardised and transformed to the **log-magnitude FFT**:

```
X(f) = log(1 + |FFT{x(t)}|)
```

This compresses the dynamic range and provides a fixed-length frequency-domain input to the CNN. A time-domain model was also trained for comparison.

### Model Architecture

A lightweight **1D CNN**:

| Layer | Details |
|---|---|
| Conv1D | 16 filters, kernel size 7, ReLU, same padding |
| MaxPool1D | pool size 2 |
| GlobalAveragePooling1D | — |
| Dense | 1 unit, sigmoid output |

Trained with Adam (lr=1e-3), binary cross-entropy loss, 50 epochs, batch size 256. Decision threshold set to the 95th percentile of negative training scores to target FPR = 0.05.

---

## Results

Performance evaluated across 9 SNR bins (5–50), with 1000 examples per class per bin:

| SNR bin | FPR (log-FFT) | Accuracy |
|---|---|---|
| 5–10  | 0.037 | 0.505 |
| 10–15 | 0.030 | 0.511 |
| 15–20 | 0.039 | 0.497 |
| 20–25 | 0.043 | 0.498 |
| 25–30 | 0.040 | 0.502 |
| 30–35 | 0.039 | 0.500 |
| 35–40 | 0.034 | 0.507 |
| 40–45 | 0.040 | 0.506 |
| 45–50 | 0.048 | 0.495 |

The FPR is controlled stably below the 0.05 target across all SNR bins. Accuracy near chance (~0.50) indicates the log-FFT representation discards phase/timing information that distinguishes blips from WNBs. 

At higher SNR (10–100 for WNB, 10–20 for glitch), the model achieves **AUC ~96.8%** with FPR ~0.05 on the log-FFT representation.

### Log-FFT vs Time-Domain

| SNR Regime | Representation | AUC (%) | FPR |
|---|---|---|---|
| WNB 10–100, Glitch 10–20 | Log-FFT | **96.8** | 0.05 |
| WNB 10–100, Glitch 10–20 | Time-domain | 88.0 | — |
| WNB 5–20, Glitch 5–20 | Log-FFT | 70.0 | 0.12 |
| WNB 5–20, Glitch 5–20 | Time-domain | **75.0** | **0.005** |

At low SNR, time-domain representation outperforms log-FFT — suggesting phase/timing features matter more when signal power is weak.
